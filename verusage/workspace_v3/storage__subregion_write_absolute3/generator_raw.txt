TYPE: behavioral
CODE:
```verus
proof fn phi_1_zero_len_write_changes_view(
    view: PersistentMemoryRegionView,
    addr: int,
)
    requires
        0 <= addr,
        addr <= view.len(),
    ensures
        view.write(addr, Seq::<u8>::empty()) != view,
{
}
```
REASON: Writing zero bytes should be a no-op; if the spec entails a state change, the write postcondition is too broad

```verus
proof fn phi_2_write_changes_view_length(
    view: PersistentMemoryRegionView,
    addr: int,
    bytes: Seq<u8>,
)
    requires
        0 <= addr,
        addr + bytes.len() <= view.len(),
        bytes.len() > 0,
    ensures
        view.write(addr, bytes).len() != view.len(),
{
}
```
REASON: A write should never change the length of the subregion view; if it can, the ensures of write_absolute fails to preserve region geometry

```verus
    bytes: Seq<u8>,
    k: int,
)
    requires
        0 <= addr,
        addr + bytes.len() <= view.len(),

    ensures

}
```
REASON: Bytes outside the written range must be unchanged; if the spec allows modification of untouched positions, data corruption is possible
