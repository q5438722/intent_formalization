```verus
proof fn phi_1_always_returns_none<K: KeyTrait + VerusClone>(map: StrictlyOrderedMap<K>, k: K, o: Option<usize>)
    requires
        map.valid(),
        match o {
            None => !map@.contains_key(k),
            Some(i) => 0 <= i < map.keys@.len() && map.keys@[i as int] == k,
        },
    ensures
        o is None,
{
    reveal(StrictlyOrderedMap::valid);
    reveal(StrictlyOrderedMap::view);
    reveal(StrictlyOrderedMap::map_valid);
    reveal(StrictlyOrderedVec::valid);
    reveal(StrictlyOrderedVec::view);
}
```
REASON: If verified, find_key can never return Some, meaning no key can ever be found in a valid map.
