NAME: write_immediately_persists
TARGET_FN: write_relative
TYPE: behavioral
CODE:
```verus
proof fn phi_1_write_immediately_persists(
    view: PersistentMemoryRegionView,
    addr: int,
    bytes: Seq<u8>,
)
    requires
        0 <= addr,
        bytes.len() > 0,
        addr + bytes.len() <= view.len(),
        view.no_outstanding_writes_in_range(addr, addr + bytes.len()),
        view.state[addr].state_at_last_flush != bytes[0 as int],
    ensures
        view.write(addr, bytes).state[addr].state_at_last_flush == bytes[0 as int],
{
}
```
REASON: write should only set outstanding_write, not immediately update state_at_last_flush (that requires a flush)
===PHI_END===

NAME: empty_write_alters_view
TARGET_FN: write_relative
TYPE: boundary
CODE:
```verus
proof fn phi_2_empty_write_alters_view(
    view: PersistentMemoryRegionView,
    addr: int,
)
    requires
        0 <= addr,
        addr <= view.len(),
        view.len() > 0,
    ensures
        view.write(addr, Seq::<u8>::empty()) =~= view ==> false,
{
}
```
REASON: Writing zero bytes should be a no-op; if it changes the view, the write spec is unsound
===PHI_END===
