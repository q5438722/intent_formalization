```verus
proof fn phi_4_io_mapping_leaks_to_other(old_self: PageAllocator, new_self: PageAllocator, ret: PagePtr, p: PagePtr, ioid: IOid, va: VAddr)
    requires
        old_self.wf(),
        new_self.wf(),
        p != ret,
        old_self.page_is_mapped(p),
        new_self.page_is_mapped(p),
        !old_self.page_io_mappings(p).contains((ioid, va)),
        new_self.page_io_mappings(ret) =~= Set::<(IOid, VAddr)>::empty().insert((ioid, va)),
        forall|q: PagePtr|
            #![trigger new_self.page_is_mapped(q)]
            #![trigger new_self.page_io_mappings(q)]
            new_self.page_is_mapped(q) && q != ret ==> new_self.page_io_mappings(q) =~= old_self.page_io_mappings(q),
    ensures
        new_self.page_io_mappings(p).contains((ioid, va)),
{
}
```
REASON: The new IO mapping (ioid, va) must not leak to any other already-mapped page; this would violate IO isolation between pages
