NAME: entry_size_equal_across_layers
TARGET_FN: Arch::entry_size_is_next_layer_size
TYPE: boundary
CODE:
```verus
proof fn phi_5_entry_size_equal_across_layers()
    ensures
        exists|a: Arch| a.inv() && a.layers.len() >= 2
            && a.entry_size(0) == a.entry_size(1),
{
    let layer = ArchLayer { entry_size: 1, num_entries: 1 };
    let a = Arch { layers: seq![layer, layer] };
    assert(a.entry_size(0) == 1);
    assert(a.entry_size(1) == 1);
    assert(a.num_entries(1) == 1);
}
```
REASON: Two consecutive layers with identical entry_size (via num_entries=1) means a layer adds no address-space subdivision, violating the x86 expectation that each layer partitions into multiple entries
