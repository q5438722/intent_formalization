NAME: write_changes_subregion_length
TARGET_FN: write_absolute
TYPE: behavioral
CODE:
```verus
proof fn phi_1_write_changes_subregion_length(
    view_before: PersistentMemoryRegionView,
    addr: int,
    bytes: Seq<u8>,
)
    requires
        0 <= addr,
        addr + bytes.len() <= view_before.len(),
        view_before.no_outstanding_writes_in_range(addr, addr + bytes.len()),
    ensures
        view_before.write(addr, bytes).len() != view_before.len(),
{
}
```
REASON: Write should never change the length of the subregion view; if it did, invariant `self.view(wrpm).len() == self.len()` would break on subsequent calls.
===PHI_END===

NAME: empty_write_changes_view
TARGET_FN: write_absolute
TYPE: boundary
CODE:
```verus
proof fn phi_2_empty_write_changes_view(
    view_before: PersistentMemoryRegionView,
    addr: int,
)
    requires
        view_before.len() > 0,
        0 <= addr,
        addr <= view_before.len(),
        view_before.no_outstanding_writes_in_range(addr, addr),
    ensures
        view_before.write(addr, Seq::<u8>::empty()) != view_before,
{
}
```
REASON: A zero-length write must be a no-op; if the spec allowed it to modify the view, callers could observe spurious state changes from vacuous writes.
===PHI_END===

proof fn phi_3_write_modifies_outside_range(
    view_before: PersistentMemoryRegionView,
    addr: int,
    bytes: Seq<u8>,
    k: int,
)
    requires
        0 <= addr,
        addr + bytes.len() <= view_before.len(),
        0 <= k < view_before.len(),
        !(addr <= k < addr + bytes.len()),
        view_before.no_outstanding_writes_in_range(addr, addr + bytes.len()),
    ensures
        view_before.write(addr, bytes).state[k] != view_before.state[k],
{
}
```
REASON: Bytes outside the written range must be unaffected; if write could modify them, the `views_differ_only_where_subregion_allows` invariant would be violated.
===PHI_END===

)
    requires
        0 <= addr,

{
}
```
REASON: Write must only set outstanding_write, never alter state_at_last_flush; changing it would corrupt the crash-consistency model since flush recovery depends on state_at_last_flush being stable.

REASON: A non-empty write to a region with no outstanding writes must visibly change the view (outstanding_write goes from None to Some); if it were a no-op the write would be silently lost.
