```verus
proof fn phi_1_serialize_noop(val: u64, old_data: Seq<u8>, new_data: Seq<u8>)
    requires
        val.is_marshalable(),
        new_data.len() >= old_data.len(),
        new_data.subrange(0, old_data.len() as int) == old_data,
        new_data.subrange(old_data.len() as int, new_data.len() as int) == val.ghost_serialize(),
    ensures
        new_data.len() == old_data.len(),
{
}
```
REASON: Serialize of a u64 should always append bytes; a no-op (zero-length append) would mean the encoding is empty and undecodable.

```verus
proof fn phi_4_serialize_constant_output(val: u64)
    requires
        val.is_marshalable(),
        val != 0u64,
    ensures
        val.ghost_serialize() == (0u64).ghost_serialize(),
{
}
```
REASON: If every non-zero u64 serialized identically to zero, the encoding would carry no information and all values would be conflated.
