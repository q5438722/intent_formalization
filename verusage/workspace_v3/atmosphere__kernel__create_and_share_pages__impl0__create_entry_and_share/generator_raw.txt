TARGET_FN: create_entry_and_share
TYPE: behavioral
CODE:
```verus
proof fn phi_1_share_flips_write_perm(
    old_src_space: Map<VAddr, MapEntry>,
    old_target_space: Map<VAddr, MapEntry>,
    new_target_space: Map<VAddr, MapEntry>,
    src_va: VAddr,
    target_va: VAddr,
)
    requires
        old_src_space.dom().contains(src_va),
        !old_target_space.dom().contains(target_va),
        new_target_space =~= old_target_space.insert(target_va, old_src_space[src_va]),
    ensures
        new_target_space[target_va].write != old_src_space[src_va].write,
{
}
```
REASON: If sharing flips write permissions, a read-only page could become writable in the target, breaking memory safety
