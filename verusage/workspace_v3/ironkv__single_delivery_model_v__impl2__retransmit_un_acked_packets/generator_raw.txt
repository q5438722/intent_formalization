```verus
proof fn phi_1_retransmit_always_empty(sd: CSingleDelivery, src: EndPoint, packets: Seq<CPacket>)
    requires
        sd.valid(),
        src.abstractable(),
        abstractify_seq_of_cpackets_to_set_of_sht_packets(packets) == sd@.un_acked_messages(src@),
        outbound_packet_seq_is_valid(packets),
        outbound_packet_seq_has_correct_srcs(packets, src@),
        sd@.un_acked_messages(src@) == packets.map_values(|p: CPacket| p@).to_set(),
        CSingleDelivery::packets_are_valid_messages(packets),
    ensures
        packets.len() == 0,
{
}
```
REASON: Would mean retransmit never produces any packets, making retransmission a no-op regardless of un-acked state

```verus
        sd.send_state.contains_key(dst),
        sd.send_state[dst].un_acked.len() > 0,
```

```verus
}
```
REASON: All packets in un_acked_messages_for_dest are constrained to have p.src == src; a packet with wrong src would be a spoofed origin

```verus
    requires
        sd.send_state.contains_key(dst),
        c1 <= c2,
        c2 <= sd.send_state[dst].un_acked.len(),
    ensures
        !sd.un_acked_messages_for_dest_up_to(src, dst, c1).subset_of(
            sd.un_acked_messages_for_dest_up_to(src, dst, c2)),
{
}
```
REASON: Increasing the count should only add packets; losing packets when scanning further would silently drop retransmissions
