TYPE: behavioral
CODE:
```verus
proof fn phi_3_mmap_always_returns_quota_error(os: OS, os_new: OS, thread_ptr: ThreadPtr, va_range: VaRange4K, ret: SyscallReturnStruct)
    requires
        os.total_wf(),
        os.thread_dom().contains(thread_ptr),
        va_range.wf(),
        va_range.len * 4 < usize::MAX,
        syscall_mmap_spec(os, os_new, thread_ptr, va_range, ret),
    ensures
        ret =~= SyscallReturnStruct::NoSwitchNew(RetValueType::ErrorNoQuota),
{
}
```
REASON: mmap would unconditionally fail with ErrorNoQuota regardless of actual quota availability, making memory mapping impossible
