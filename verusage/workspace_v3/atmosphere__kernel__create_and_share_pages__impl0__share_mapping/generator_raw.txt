```verus
proof fn phi_3_other_page_mapping_affected(
    old_pm: Map<PagePtr, Set<(ProcPtr, VAddr)>>,
    new_pm: Map<PagePtr, Set<(ProcPtr, VAddr)>>,
    entry_addr: PagePtr,
    other_page: PagePtr,
    target_proc_ptr: ProcPtr,
    target_va: VAddr,
)
    requires
        old_pm.dom().contains(entry_addr),
        old_pm.dom().contains(other_page),
        other_page != entry_addr,
        new_pm.dom() =~= old_pm.dom(),
        forall|page_ptr: PagePtr|
            #![trigger new_pm[page_ptr]]
            old_pm.dom().contains(page_ptr) && page_ptr != entry_addr
                ==> old_pm[page_ptr] =~= new_pm[page_ptr],
        new_pm[entry_addr] =~= old_pm[entry_addr].insert((target_proc_ptr, target_va)),
    ensures
        new_pm[other_page] != old_pm[other_page],
{
}
```
REASON: If sharing a mapping mutates the reverse-mapping set of an unrelated physical page, page tracking is corrupted and could lead to use-after-free
===PHI_END===
