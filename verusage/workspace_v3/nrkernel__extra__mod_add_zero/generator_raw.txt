NAME: all_addrs_aligned
TARGET_FN: aligned
TYPE: behavioral
CODE:
```verus
proof fn phi_1_all_addrs_aligned(addr: nat, size: nat)
    requires
        size > 0,
    ensures
        aligned(addr, size),
{
}
```
REASON: If every address is trivially aligned to any positive size, the spec is vacuously weak and useless as a constraint.

===PHI_START===
NAME: succ_preserves_alignment
TARGET_FN: aligned
TYPE: boundary
CODE:
```verus
proof fn phi_2_succ_preserves_alignment(addr: nat, size: nat)
    requires
        aligned(addr, size),
        size > 1,
    ensures
        aligned(addr + 1, size),
{
}
```
REASON: Alignment to a size > 1 must not be preserved by incrementing by 1; otherwise alignment is meaningless.
===PHI_END===

===PHI_START===
NAME: double_size_aligned
TARGET_FN: aligned
TYPE: logical
CODE:
```verus
proof fn phi_3_double_size_aligned(addr: nat, size: nat)
    requires
        aligned(addr, size),
        size > 0,
    ensures
        aligned(addr, 2 * size),
{
}
```
REASON: Being aligned to size should not automatically imply alignment to 2*size (e.g., 4 is aligned to 4 but not to 8).
===PHI_END===

NAME: only_zero_aligned
TARGET_FN: aligned
TYPE: behavioral
CODE:
```verus

    requires
        aligned(addr, size),
        size > 0,
    ensures
        addr == 0,
{
}
```
REASON: If the only aligned address were 0, the spec would be over-constrained; multiples of size must also be aligned.

NAME: aligned_implies_one_aligned
TARGET_FN: aligned
TYPE: logical
CODE:
```verus
proof fn phi_5_aligned_implies_one_aligned(size: nat)
    requires
        size > 1,
        aligned(size, size),
    ensures

}
```
REASON: 1 should never be aligned to any size > 1; if it were, the modular arithmetic in the spec would be broken.
