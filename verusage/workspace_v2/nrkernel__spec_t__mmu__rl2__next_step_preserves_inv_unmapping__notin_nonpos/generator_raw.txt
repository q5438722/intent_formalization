NAME: from_bits_supervisor_inverted
TYPE: behavioral
CODE:
```verus
proof fn phi_4_from_bits_supervisor_inverted()
    ensures
        Flags::from_bits(true, true, false).is_supervisor == false,
        Flags::from_bits(true, false, false).is_supervisor == true,
{
}
```
REASON: `from_bits` sets `is_supervisor: !flag_US` — so US=true means not-supervisor and US=false means supervisor; this inversion matches x86 semantics where the US bit grants user access, but if inverted incorrectly, kernel pages would be accessible from userspace

```verus
        mem1.pml4 == mem2.pml4,
        mem1.read(add(mem1.pml4, mul(l0_bits!(vaddr), WORD_SIZE))) & 1 == 0,

    ensures
        mem1.pt_walk(vaddr).path == mem2.pt_walk(vaddr).path,
{
}
```
REASON: If two memories agree on pml4 and the L0 entry (which is non-present), the walk should terminate at L0 with the same path — tests that pt_walk short-circuits on Invalid entries and doesn't read further memory
