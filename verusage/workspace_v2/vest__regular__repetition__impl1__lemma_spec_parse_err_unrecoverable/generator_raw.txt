```verus
proof fn phi_3_error_propagates_upward<C: SecureSpecCombinator>(c: &RepeatN<C>, s: Seq<u8>, n1: usize, n2: usize)
    requires
        n1 <= n2,
        RepeatN(c.0, n1).spec_parse(s).is_none(),
    ensures
        RepeatN(c.0, n2).spec_parse(s).is_none(),
{
    let c1 = RepeatN(c.0, n1);
    let c2 = RepeatN(c.0, n2);
    c1.lemma_spec_parse_err_unrecoverable(s, n1, n2);
}
```
REASON: If parsing fails at repetition count n1, it fails for all larger counts — this monotonicity is proven but depends on the closed spec; if the sequential parsing order had a bug, the lemma would still hold vacuously
