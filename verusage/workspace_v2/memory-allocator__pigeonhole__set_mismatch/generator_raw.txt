```verus
proof fn phi_2_set_mismatch_derives_false(s1: Set<nat>, s2: Set<nat>, missing: nat)
    requires
        s1.finite(),
        s2.finite(),
        s1.len() == s2.len(),
        forall |elt| s2.contains(elt) ==> s1.contains(elt),
        s1.contains(missing),
        !s2.contains(missing),
    ensures
        false,
{
    set_mismatch(s1, s2, missing);
}
```
REASON: `set_mismatch` derives `false` — a contradiction lemma whose correctness depends on the external_body `singleton_set_unique_elt`; the chain of trust bottoms out at an unverified axiom

TYPE: behavioral

```verus
proof fn phi_3_singleton_any_two_elements_equal()
    ensures ({
        let s = Set::<nat>::empty().insert(5);
        forall |a: nat, b: nat| s.contains(a) && s.contains(b) ==> a == b
    }),
{
    let s = Set::<nat>::empty().insert(5);
    assert forall |a: nat, b: nat| s.contains(a) && s.contains(b) implies a == b by {
        assert(s.finite());
        assert(s.len() == 1);
        singleton_set_unique_elt(s, a, b);
    }
}
```
REASON: The external_body axiom allows deriving that any concrete singleton set has exactly one element — testing the axiom on a concrete set to confirm it doesn't over-constrain
