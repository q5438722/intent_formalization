NAME: pde_layer0_never_page
TYPE: behavioral
CODE:
```verus
proof fn phi_1_pde_layer0_never_page(entry: usize)
    requires
        entry & 1 == 1,
    ensures
        !((PDE { entry, layer: Ghost(0) })@ is Page),
{
}
```
REASON: Layer 0 (PML4) entries should never be Page — they are always Directory or Invalid; if a Page entry appears at layer 0, the walk semantics would be wrong

NAME: walk_result_vbase_leq_vaddr
TYPE: behavioral
CODE:
```verus
proof fn phi_3_walk_result_vbase_leq_vaddr(w: Walk)
    requires
        w.path.last().1 is Page,
        w.path.len() >= 2,
        w.path.len() <= 4,
    ensures
        w.result() is Valid,
        w.result()->vbase <= w.vaddr,
{
}
```
REASON: `vbase = align_to_usize(vaddr, entry_size)` rounds down, so `vbase <= vaddr` — if this fails, the alignment function is incorrectly rounding up
===PHI_END===
