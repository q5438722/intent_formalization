```verus
proof fn phi_4_unmarshal_wrong_kind_succeeds<T: ResourceView>(obj: DynamicObjectView)
    requires
        obj.kind != T::kind(),
        T::unmarshal_spec(obj.spec) is Ok,
        T::unmarshal_status(obj.status) is Ok,
    ensures
        T::unmarshal(obj) is Ok,
{
}
```
REASON: unmarshal explicitly rejects kind mismatches as its first check — valid spec and status should NOT override a wrong kind
