NAME: map_min_value_ignores_keys_beyond_idx

```verus
proof fn phi_4_map_min_value_ignores_keys_beyond_idx(m: Map<NodeId, nat>, idx: nat, extra_val: nat)
    requires
        forall|i| 0 <= i <= idx ==> m.contains_key(i),
        !m.contains_key(idx + 1),
        extra_val < map_min_value(m, idx),
    ensures
        map_min_value(m, idx) == map_min_value(m.insert(idx + 1, extra_val), idx),
{
}
```
REASON: `map_min_value(m, idx)` only examines keys `[0, idx]` — inserting a smaller value at `idx+1` doesn't change the result, so the "minimum" over the map can miss the actual global minimum
===PHI_END===
