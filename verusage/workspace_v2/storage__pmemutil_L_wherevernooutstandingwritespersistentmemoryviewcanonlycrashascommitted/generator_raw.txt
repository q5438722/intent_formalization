```verus
proof fn phi_4_two_addrs_same_chunk_same_outcome(
    view: PersistentMemoryRegionView,
    s: Seq<u8>,
    a: int,
    b: int,
)
    requires
        view.can_crash_as(s),
        0 <= a < view.len(),
        0 <= b < view.len(),
        a / const_persistence_chunk_size() == b / const_persistence_chunk_size(),
        view.state[a].outstanding_write.is_none(),
        view.state[b].outstanding_write.is_none(),
    ensures
        s[a] == view.state[a].state_at_last_flush,
        s[b] == view.state[b].state_at_last_flush,
{
    lemma_if_no_outstanding_writes_at_addr_then_persistent_memory_view_can_only_crash_as_committed(view, a);
    lemma_if_no_outstanding_writes_at_addr_then_persistent_memory_view_can_only_crash_as_committed(view, b);
}
```
REASON: Two addresses in the same chunk with no outstanding writes must both crash as committed — this is a corollary of the per-address external_body lemma, testing that the trusted assumption composes correctly across addresses
===PHI_END===
