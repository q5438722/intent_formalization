NAME: flatten_order_independent
TYPE: behavioral
CODE:
```verus
proof fn phi_3_flatten_order_independent(s1: Set<int>, s2: Set<int>)
    ensures
        flatten_set_seq(seq![s1, s2]) == flatten_set_seq(seq![s2, s1]),
{
    lemma_flatten_set_seq_spec(seq![s1, s2]);
    lemma_flatten_set_seq_spec(seq![s2, s1]);
    assert(flatten_set_seq(seq![s1, s2]) =~= flatten_set_seq(seq![s2, s1]));
}
```
REASON: If flatten depended on ordering, the set union semantics would be broken — `fold_left` with union should be commutative

REASON: If duplicating a set in the sequence enlarged the result, union idempotency would be violated

NAME: flatten_subset_of_append
TYPE: logical
CODE:
```verus
proof fn phi_5_flatten_subset_of_append(sets: Seq<Set<int>>, extra: Set<int>)
    ensures
        flatten_set_seq(sets).subset_of(flatten_set_seq(sets.push(extra))),
{
    lemma_flatten_set_seq_spec(sets);
    lemma_flatten_set_seq_spec(sets.push(extra));
    assert forall |x: int| flatten_set_seq(sets).contains(x) implies flatten_set_seq(sets.push(extra)).contains(x) by {
        let i = choose |i: int| 0 <= i < sets.len() && sets[i].contains(x);
        assert(sets.push(extra)[i].contains(x));
    }
}
```
REASON: If appending a set could shrink the flattened result, the union-based fold_left would be losing elements
