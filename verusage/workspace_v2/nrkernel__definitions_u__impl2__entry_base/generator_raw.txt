```verus
proof fn phi_3_entry_base_idx_equals_num_entries(arch: ArchExec, layer: usize, base: usize)

        arch@.inv(),
        layer < arch@.layers.len(),
        base <= MAX_BASE,
    ensures
        arch@.entry_base(layer as nat, base as nat, X86_NUM_ENTRIES as nat) <= MAX_BASE + X86_MAX_ENTRY_SIZE * X86_NUM_ENTRIES as nat,
{
    mult_leq_mono_both(X86_NUM_ENTRIES as nat, arch@.entry_size(layer as nat), X86_NUM_ENTRIES as nat, X86_MAX_ENTRY_SIZE);

```
REASON: `idx` can equal `X86_NUM_ENTRIES` (512), not just be less than it — the precondition allows one-past-the-end index, producing an address potentially beyond the mapped region
