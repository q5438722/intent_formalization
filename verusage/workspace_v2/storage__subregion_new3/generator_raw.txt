```verus
proof fn phi_1_subregion_view_length_matches(
    region: PersistentMemoryRegionView,
    start: nat,
    len: nat,
)
    requires
        start + len <= region.len(),
    ensures
        get_subregion_view(region, start, len).len() == len,
{
}
```
REASON: The subregion view length should equal the requested length — if `subrange` produced a different length, all offset calculations within the subregion would be invalid

NAME: views_differ_allows_writable_addr_change
TYPE: logical
CODE:
```verus
proof fn phi_3_views_differ_allows_writable_addr_change(
    v1: PersistentMemoryRegionView,
    v2: PersistentMemoryRegionView,
    start: nat,
    len: nat,
    addr: int,
)
    requires
        v1.len() == v2.len(),
        start + len <= v1.len(),
        start <= addr < start + len,
        forall |a: int| a != addr && (
            ||| 0 <= a < start
            ||| start + len <= a < v1.len()
            ||| start <= a < start + len
        ) ==> v1.state[a] == v2.state[a],
    ensures
        views_differ_only_where_subregion_allows(v1, v2, start, len, |a: int| a == addr),
{
}
```
REASON: When only one address is writable and only that address differs, the predicate should hold — tests that the writable address function correctly permits targeted mutations
===PHI_END===

```verus
)

        start <= region.len(),
    ensures
        get_subregion_view(region, start, 0).len() == 0,
```

REASON: A subregion starting at `start+len1` should align with the second half of a combined subregion — if subrange arithmetic were inconsistent, splitting a region into adjacent subregions would produce misaligned views
