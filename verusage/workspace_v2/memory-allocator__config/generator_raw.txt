NAME: slice_shift_mismatch
TYPE: logical
CODE:
```verus
proof fn phi_1_slice_shift_mismatch()
    ensures
        SLICE_SIZE == (1u64 << SLICE_SHIFT),
{
    assert(1u64 << SLICE_SHIFT == 65536) by (compute);
}
```
REASON: `SLICE_SIZE` is hardcoded as 65536 with a comment `(1 << SLICE_SHIFT)` but `SLICE_SHIFT = 13 + 3 = 16` and `1 << 16 = 65536` — if the shift constant were ever changed without updating the hardcoded value, they'd silently diverge

proof fn phi_5_medium_obj_wsize_max_truncation()

        MEDIUM_OBJ_WSIZE_MAX * (usize::BITS as u64 / 8) == MEDIUM_OBJ_SIZE_MAX,

    assert(MEDIUM_OBJ_WSIZE_MAX == 16384u64) by (compute_only);
