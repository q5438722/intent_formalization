```verus
proof fn phi_3_next_entry_base_off_by_one(arch: Arch, layer: nat, base: nat, idx: nat)
    requires
        arch.inv(),
        layer < arch.layers.len(),
    ensures
        arch.next_entry_base(layer, base, idx) == base + idx * arch.entry_size(layer) + arch.entry_size(layer),
{
}
```
REASON: `next_entry_base` computes `base + (idx+1)*entry_size` which equals `entry_base(idx) + entry_size` — it returns the END of the idx-th entry, not the START of the (idx+1)-th, but these are the same value, potentially confusing callers about semantics
