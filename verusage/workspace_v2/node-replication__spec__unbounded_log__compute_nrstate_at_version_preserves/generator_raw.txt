NAME: preserves_allows_extra_keys
TYPE: logical
CODE:
```verus
proof fn phi_3_preserves_allows_extra_keys<DT: Dispatch>(
    a: Map<LogIdx, LogEntry<DT>>,
    b: Map<LogIdx, LogEntry<DT>>,
    extra_entry: LogEntry<DT>,
    version: LogIdx,
)
    requires
        forall |i| 0 <= i < version ==> a.contains_key(i),
        forall |i| 0 <= i < version ==> a[i] == b[i],
        !a.contains_key(version),
        b == a.insert(version, extra_entry),
    ensures
        compute_nrstate_at_version::<DT>(a, version) == compute_nrstate_at_version::<DT>(b, version),
{
    compute_nrstate_at_version_preserves(a, b, version);
}
```
REASON: Maps that agree on `[0, version)` produce the same state even if `b` has extra entries beyond `version` — the preservation lemma ignores entries at or above the version
===PHI_END===

NAME: compute_nrstate_deterministic_same_ops
TYPE: behavioral
CODE:
```verus
proof fn phi_4_compute_nrstate_deterministic_same_ops<DT: Dispatch>(
    log1: Map<LogIdx, LogEntry<DT>>,
    log2: Map<LogIdx, LogEntry<DT>>,
    version: LogIdx,
)
    requires
        forall |i| 0 <= i < version ==> log1.contains_key(i),
        forall |i| 0 <= i < version ==> log2.contains_key(i),
        forall |i: nat| i < version ==> log1[i].op === log2[i].op && log1[i].node_id == log2[i].node_id,
    ensures
        compute_nrstate_at_version::<DT>(log1, version) == compute_nrstate_at_version::<DT>(log2, version),
{
    compute_nrstate_at_version_preserves(log1, log2, version);
}
```
REASON: State computation depends only on the op sequence, not the log map identity — two different maps with identical entries produce the same state, meaning `node_id` is never used in state computation
===PHI_END===
