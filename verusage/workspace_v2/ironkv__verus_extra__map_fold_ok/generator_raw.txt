```verus
proof fn phi_4_map_fold_singleton(x: int, f: spec_fn(int) -> int)
    ensures
        map_fold(set![x], f) == set![f(x)],
{
    map_fold_ok(set![x], f);
    assert(set![x].map(f) =~= set![f(x)]);
}
```
REASON: If mapping a singleton set didn't produce a singleton with the mapped element, set_fold's choose+remove recursion would be broken at the simplest case
