NAME: interp_log_ignores_extra_entries
TYPE: logical
CODE:
```verus
proof fn phi_3_interp_log_ignores_extra_entries<DT: Dispatch>(
    i_log: Map<LogIdx, LogEntry<DT>>,
    extra: LogEntry<DT>,
    gtail: nat,
)
    requires
        forall|i| 0 <= i < gtail ==> i_log.contains_key(i),
        !i_log.contains_key(gtail),
    ensures
        interp_log::<DT>(gtail, i_log) == interp_log::<DT>(gtail, i_log.insert(gtail, extra)),
{
    assert(interp_log::<DT>(gtail, i_log) =~= interp_log::<DT>(gtail, i_log.insert(gtail, extra)));
}
```
REASON: Entries at or beyond `gtail` are invisible to `interp_log` — the map-to-seq interpretation silently truncates, so appending entries doesn't change the interpreted log
===PHI_END===
