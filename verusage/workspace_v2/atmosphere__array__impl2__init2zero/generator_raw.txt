```verus
        old_arr.wf(),
        0 <= i < 5,
        new_arr.seq@ =~= old_arr.seq@.update(i as int, v),
        new_arr.wf(),
    ensures
        new_arr.seq@.len() == old_arr.seq@.len(),
{
}
```
REASON: set's ensures guarantee wf() on the output, but the length preservation is derived from Seq::update semantics — if update had a different axiomatization, wf() alone wouldn't catch length changes
