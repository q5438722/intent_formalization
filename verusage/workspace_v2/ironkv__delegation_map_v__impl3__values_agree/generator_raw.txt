```verus
}
```
REASON: The second return value `ret.1` represents "almost agrees" (all match except the last) — but when `ret.0` is false and `ret.1` is true, the hi element disagrees while all prior agree; when `ret.1` is false, some element before hi also disagrees, but the spec doesn't tell you which one
