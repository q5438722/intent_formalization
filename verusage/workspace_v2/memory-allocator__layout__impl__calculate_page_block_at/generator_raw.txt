```verus
proof fn phi_5_calculate_overflow_not_checked(page_id: PageId, block_size: usize, idx: usize)
    requires
        block_start_at(page_id, block_size as int, 0) + idx as int * block_size as int
            <= segment_start(page_id.segment_id) + SEGMENT_SIZE,
        segment_start(page_id.segment_id) + SEGMENT_SIZE < usize::MAX,
        block_start_at(page_id, block_size as int, 0) == page_start(page_id) + start_offset(block_size as int),
    ensures
        block_start_at(page_id, block_size as int, idx as int) <= segment_start(page_id.segment_id) + SEGMENT_SIZE,

    assert(idx as int * block_size as int == block_size as int * idx as int) by(nonlinear_arith);
}
```
REASON: The result of `calculate_page_block_at` is bounded by `segment_start + SEGMENT_SIZE < usize::MAX` — but there's no check that `page_start + block_size * idx` doesn't overflow `usize` in the intermediate computation; the precondition ensures the mathematical result fits, relying on Verus's overflow checking
