REASON: The same empty range is simultaneously consistent with two different endpoints — `range_consistent` provides no discrimination for degenerate intervals, so callers cannot deduce endpoint uniqueness from consistency claims

proof fn phi_3_end_iterator_geq_everything<K: KeyTrait + VerusClone>(dm: DelegationMap<K>, ki: KeyIterator<K>, id: ID)
    requires
        ki.k.is_None(),
    ensures
        dm.range_consistent(&ki, &KeyIterator::new_spec(K::zero_spec()), &id),
{
    K::cmp_properties();
    dm.empty_key_range_is_consistent(&ki, &KeyIterator::new_spec(K::zero_spec()), &id);
}
```
REASON: End iterator (None) is `geq` everything because `lt_spec` requires `!self.k.is_None()`, so `geq_spec` is trivially true — making any range from end to anywhere vacuously consistent regardless of actual delegation

proof fn phi_5_range_consistent_contradicts_map_value<K: KeyTrait + VerusClone>(dm: DelegationMap<K>, lo: K, hi: K, k: K, id: ID)
    requires
        lo.cmp_spec(k).lt(),
        k.cmp_spec(hi).lt(),
        dm@[k] != id@,
        lo.cmp_spec(hi).lt(),
    ensures
        !dm.range_consistent(&KeyIterator::new_spec(lo), &KeyIterator::new_spec(hi), &id),
{
    K::cmp_properties();
    assert(KeyIterator::between(KeyIterator::new_spec(lo), KeyIterator::new_spec(k), KeyIterator::new_spec(hi)));
}
```
REASON: If a key strictly between lo and hi maps to a different endpoint, range_consistent must be false — verifying this confirms that range_consistent is not vacuously true for proper intervals (a sanity check that the spec is not trivially weak)
