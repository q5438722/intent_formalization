REASON: For small sizes like 4, `u64_leading_zeros(4) == 61`, so `b = 63 - 61 = 2`, and `(2 << 2) | ((4 >> 0) & 0x03) = 8 | 0 = 8 >= 4` should hold — but if it doesn't, the property excludes valid small sizes from the sbin computation
