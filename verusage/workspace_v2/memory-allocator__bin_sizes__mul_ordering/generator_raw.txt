NAME: div2_no_upper_bound_on_x
TYPE: boundary
CODE:
```verus
proof fn phi_3_div2_no_upper_bound_on_x(y: int)
    requires
        y > 0,
    ensures
        u64::MAX as int / (y * 2) == (u64::MAX as int / y) / 2,
{
    div2(u64::MAX, y);
}
```
REASON: `div2` takes `x: u64` but `y: int` with no upper bound — `y * 2` can exceed u64 range, making the lemma applicable to divisions by values that don't fit in machine words

NAME: mul_assoc_smuggles_commutativity
TYPE: logical
CODE:
```verus
proof fn phi_4_mul_assoc_smuggles_commutativity(x: nat, y: nat)
    ensures
        x * y == y * x,
{
    mul_assoc(x, y, 1);
}
```
REASON: `mul_assoc` ensures `(x*y)*z == y*(x*z)` which encodes commutativity — setting z=1 directly yields `x*y == y*x`, which may let proofs go through for unintended reasons
