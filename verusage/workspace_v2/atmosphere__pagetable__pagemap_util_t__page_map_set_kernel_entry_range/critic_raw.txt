REASONING: The external_body ensures both `wf()` (which requires `usize2page_entry(ar@[i]) =~= spec_seq@[i]` for all i) and `value[index] =~= value`. This implicitly assumes every PageEntry is the image of some usize under `spec_usize2page_entry`, which is not true for arbitrary PageEntry values (e.g., addr with non-MEM_MASK-aligned bits). This is a real soundness gap in the trusted axiom.

===SUMMARY===
Two true positives were identified, both stemming from `page_map_set_no_requires` being an external_body with overly strong ensures. It accepts arbitrary PageEntry values without MEM_valid checks, and its simultaneous guarantee of wf() plus exact value storage implicitly assumes every PageEntry is representable as a usize — a non-trivial assumption that is never verified. The kernel entry overwrite and spec_index-without-wf candidates are false positives reflecting intended design choices.
