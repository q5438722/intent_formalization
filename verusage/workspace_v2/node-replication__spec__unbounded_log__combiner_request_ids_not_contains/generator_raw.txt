CODE:
```verus
proof fn phi_1_fresh_rid_not_in_combiner_ids(
    combiners: Map<NodeId, CombinerState>,
    rid: ReqId,
)
    requires
        combiners.dom().finite(),
        combiner_request_id_fresh(combiners, rid),
    ensures
        !combiner_request_ids(combiners).contains(rid),
{
    combiner_request_ids_not_contains(combiners, rid);
}
```
REASON: A fresh rid (not in any combiner's queued_ops) is absent from the aggregate set — tests one direction of the biconditional proved by `combiner_request_ids_not_contains`

NAME: ready_combiner_contributes_nothing
TYPE: behavioral
CODE:
```verus
proof fn phi_5_ready_combiner_contributes_nothing(
    combiners: Map<NodeId, CombinerState>,
    nid: NodeId,
    rid: ReqId,
)
    requires
        combiners.dom().finite(),
        combiners.contains_key(nid),
        combiners[nid] is Ready,
        !combiner_request_ids(combiners.remove(nid)).contains(rid),
    ensures
        !combiner_request_ids(combiners).contains(rid),
{
    combiner_request_ids_not_contains(combiners, rid);
    combiner_request_ids_not_contains(combiners.remove(nid), rid);
}
```
REASON: A Ready combiner with empty queued_ops adds nothing to the aggregate set — tests that adding a Ready node doesn't spuriously include rids
