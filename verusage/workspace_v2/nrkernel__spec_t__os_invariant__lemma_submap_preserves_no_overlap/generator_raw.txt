TYPE: logical
CODE:
```verus
proof fn phi_1_submap_shrinks_pte_size(pt: Map<nat, PTE>, sub_pt: Map<nat, PTE>, vaddr: nat, ult_id: nat)
    requires
        sub_pt.submap_of(pt),
        pt.contains_key(vaddr),
        !sub_pt.contains_key(vaddr),
    ensures
        os::CoreState::UnmapWaiting { ult_id, vaddr }.pte_size(sub_pt) == 0,
        os::CoreState::UnmapWaiting { ult_id, vaddr }.pte_size(pt) == pt[vaddr].frame.size,
{
}
```
REASON: When a key is in pt but not sub_pt, UnmapWaiting's pte_size drops to 0 — the lemma exploits this monotonicity (smaller regions overlap less), but the zero-size region means the core's operation becomes invisible to overlap checks on the submap

REASON: Two cores with identical MapWaiting states violate uniqueness — tests that `unique_CoreStates` correctly rejects duplicate non-Idle entries; if it accepted them, the no-overlap argument would break
