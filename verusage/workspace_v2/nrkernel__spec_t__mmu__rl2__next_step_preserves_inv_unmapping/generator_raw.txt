REASON: Layer 1 (PDPT) Page entries use `MASK_L1_PG_ADDR` (bits 30+) instead of `MASK_ADDR` (bits 12+) — if wrong, 1GB pages would have incorrect physical base addresses

TYPE: logical
CODE:
```verus

        layer < 4,
        entry & 1 == 1,

    ensures
        (PDE { entry, layer: Ghost(layer) })@ is Invalid,
{
}
```
REASON: When `all_mb0_bits_are_zero` returns false for a present entry, PDE::view falls through to Invalid — but since `all_mb0_bits_are_zero` is `external_body`, Verus cannot verify it actually checks the right bits, potentially misclassifying valid entries
