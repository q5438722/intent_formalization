proof fn phi_1_segment_start_mult_segment_size(segment_id: SegmentId)

NAME: segment_start_no_overlap_guarantee
TYPE: logical
CODE:
```verus
proof fn phi_3_segment_start_no_overlap_guarantee(s1: SegmentId, s2: SegmentId)
    requires
        s1.id != s2.id,
    ensures
        segment_start(s1) != segment_start(s2),
{
    assert(SEGMENT_SIZE == 33554432) by (compute);
    assert(SEGMENT_SIZE > 0);
}
```
REASON: Distinct segment IDs always map to distinct start addresses — but two `SegmentId`s with different `provenance` or `uniq` but same `id` would share the same start address while being semantically different segments

```verus
proof fn phi_4_segment_start_unbounded(segment_id: SegmentId)
    requires
        segment_id.id == 1000000000,
    ensures
        segment_start(segment_id) == 1000000000 * SEGMENT_SIZE as int,
        segment_start(segment_id) > usize::MAX as int,
{
    assert(SEGMENT_SIZE == 33554432) by (compute);
    assert(1000000000 * 33554432 > 18446744073709551615int);
}
```
REASON: `segment_id.id` is unbounded `nat` — large IDs produce start addresses exceeding `usize::MAX`, making physical addressing impossible; no constraint prevents this
===PHI_END===
