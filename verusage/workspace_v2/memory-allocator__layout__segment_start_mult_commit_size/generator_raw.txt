NAME: distinct_ids_distinct_starts
TYPE: logical
CODE:
```verus
proof fn phi_4_distinct_ids_distinct_starts(s1: SegmentId, s2: SegmentId)
    requires
        s1.id != s2.id,
    ensures
        segment_start(s1) != segment_start(s2),
{
    assert(SEGMENT_SIZE > 0) by (compute);
}
```
REASON: Different segment IDs always yield different start addresses — but `SegmentId`s with same `id` but different `provenance`/`uniq` are semantically distinct segments sharing the same address
