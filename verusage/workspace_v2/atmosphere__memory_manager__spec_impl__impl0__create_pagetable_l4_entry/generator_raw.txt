NAME: pci_bitmap_wf_domain_exact_size
TYPE: boundary
CODE:
```verus
proof fn phi_3_pci_bitmap_wf_domain_exact_size(bm: PCIBitMap, ioid: IOid, bus: u8, dev: u8, fun: u8)
    requires
        bm.wf(),
        bm.ghost_map@.dom().contains((ioid, bus, dev, fun)),
    ensures
        0 <= ioid < IOID_MAX,
        0 <= bus < 256,
        0 <= dev < 32,
        0 <= fun < 8,
{
}
```
REASON: PCIBitMap::wf uses biconditional so domain membership implies bounds — but u8 already constrains bus < 256, dev < 32, fun < 8 are NOT implied by u8 type alone, so wf silently accepts out-of-spec device/function numbers if ghost_map is constructed carelessly
===PHI_END===
