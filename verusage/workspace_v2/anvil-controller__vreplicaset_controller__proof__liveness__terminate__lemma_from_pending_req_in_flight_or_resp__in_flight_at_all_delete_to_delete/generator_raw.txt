```verus
proof fn phi_5_object_ref_ignores_uid(v1: VReplicaSetView, v2: VReplicaSetView)
    requires
        v1.metadata.name is Some,
        v2.metadata.name is Some,
        v1.metadata.namespace is Some,
        v2.metadata.namespace is Some,

}
```
REASON: object_ref uses only kind/name/namespace and ignores uid, so a deleted-and-recreated VReplicaSet with a new uid maps to the same ObjectRef — the liveness lemma's ongoing_reconciles keyed by object_ref could conflate old and new instances
