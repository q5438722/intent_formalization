```verus
    pm: &PMRegion,
    start: u64,
    len: nat,
    k: int,
)
    requires
        pm.inv(),
        start + len <= pm@.len() <= u64::MAX,
        0 <= k < len,
    ensures
        get_subregion_view(pm@, start as nat, len).state[k] == pm@.state[start as int + k],
{
}
```
REASON: Subregion byte at relative offset k should equal the region byte at absolute offset start+k — if the indexing were misaligned, reads through the subregion would access wrong physical bytes

NAME: full_region_subregion_equals_region
TYPE: boundary
CODE:
```verus
proof fn phi_4_full_region_subregion_equals_region<PMRegion: PersistentMemoryRegion>(
    pm: &PMRegion,
)
    requires
        pm.inv(),
        pm@.len() <= u64::MAX,
    ensures
        get_subregion_view(pm@, 0, pm@.len()) == pm@,
{
    assert(pm@.state.subrange(0, pm@.len() as int) =~= pm@.state);
}
```
REASON: A subregion spanning the entire region should equal the original — if it didn't, creating a full-coverage subregion would lose or alter data
===PHI_END===

REASON: The first portion of a combined subregion should match a subregion of just that portion — if they disagreed, splitting a subregion into two adjacent pieces would produce inconsistent views
