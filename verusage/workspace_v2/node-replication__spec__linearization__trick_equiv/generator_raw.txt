NAME: trick_equiv_swaps_states
TYPE: logical
CODE:
```verus
proof fn phi_3_trick_equiv_swaps_states<DT: Dispatch>(
    s1: SimpleLog::State<DT>,
    s2: SimpleLog::State<DT>,
    tail: SimpleLogBehavior<DT>,
    b: AsynchronousSingletonBehavior<DT>,
)
    requires
        behavior_equiv(
            SimpleLogBehavior::Stepped(s1, AsyncLabel::Internal, Box::new(tail)),
            b,
        ),
    ensures
        behavior_equiv(
            SimpleLogBehavior::Stepped(s2, AsyncLabel::Internal, Box::new(tail)),
            b,
        ),
{
    trick_equiv(
        SimpleLogBehavior::Stepped(s1, AsyncLabel::Internal, Box::new(tail)),
        SimpleLogBehavior::Stepped(s2, AsyncLabel::Internal, Box::new(tail)),
        b,
    );
}
```
REASON: The state in an internal step can be replaced with any other state without breaking equivalence — `trick_equiv` proves internal step states are completely irrelevant
===PHI_END===

```verus
proof fn phi_4_infinite_internal_steps_equiv<DT: Dispatch>(
    s1: SimpleLog::State<DT>,

)
    requires
        behavior_equiv(SimpleLogBehavior::Inited(s1), b),
    ensures
        behavior_equiv(
            SimpleLogBehavior::Stepped(
                s2,
                AsyncLabel::Internal,
                Box::new(SimpleLogBehavior::Stepped(
                    s2,
                    AsyncLabel::Internal,
                    Box::new(SimpleLogBehavior::Inited(s1)),
                )),
            ),
            b,
        ),
{
}
```
REASON: Arbitrarily many internal steps can be stacked on the `a` side — `behavior_equiv` peels them off recursively, so any number of unvalidated internal transitions are invisible
